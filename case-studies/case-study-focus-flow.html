01 The Problem

DISTRACTION IS THE DEFAULT

Core Problem

Modern workers spend up to 23 minutes recovering from a single interruption. Existing productivity tools are either too rigid (timers with no context) or too gamified at the expense of actual work — and none of them feel good to use.

I noticed this personally while studying and building projects. I'd open a Pomodoro timer in one tab, my task list in another, Spotify in a third — and spend more mental energy context-switching between tools than actually doing the work.

The question I kept asking: what if a single dashboard could hold your focus state, your tasks, your music, and your progress data — all in one coherent experience?

02 Research & Discovery

WHAT KEEPS PEOPLE FOCUSED

I studied productivity psychology — specifically the concepts of flow state (Csikszentmihalyi), the Zeigarnik Effect (unfinished tasks occupy mental RAM), and variable reward loops (why games are more engaging than to-do lists).

🧠
Flow State
Peak focus requires the right balance of challenge vs. skill. Too easy = boredom. Too hard = anxiety.
🎯
Clear Goals
People work better with visible, specific targets rather than open-ended "do more" instructions.
🏆
Reward Loops
Small wins (streak badges, session completion) release dopamine and reinforce the behavior.
🎵
Ambient Sound
Music with 60–70 BPM correlates with improved focus and reduced cognitive fatigue in study environments.

I also looked at existing tools: Notion, Forest, Be Focused, Todoist. None combined all four dimensions in a single, aesthetically unified space. Most felt like utilities, not experiences.

03 Design Decisions

HOW IT CAME TOGETHER

Every feature in Focus Flow was a deliberate choice to reduce friction and amplify motivation. Here are the decisions that defined the product:

01
UNIFIED DASHBOARD LAYOUT

Rather than a full-screen timer that hides everything else, I designed a dashboard that keeps the timer, task list, and stats all visible simultaneously. This reduces the need to switch contexts mentally — you see your goal and your progress at the same time.

UX Architecture
02
DRAG-AND-DROP TASK PRIORITIZATION

Tasks can be reordered by dragging. This forces the user to make an active decision about priority — which is itself a productivity technique. Implementing this with the HTML5 Drag & Drop API (no external libraries) kept the bundle light.

Interaction Design
03
SPOTIFY EMBED INTEGRATION

Rather than building a music player from scratch, I embedded Spotify's IFrame API directly in the dashboard. This gives users access to their actual playlists — not canned lo-fi tracks — while requiring zero authentication flow from my side.

API Integration
04
STREAK & GOAL SYSTEM

Completing Pomodoro sessions builds a streak. Setting and hitting daily session goals awards visual badges. This was inspired by the habit literature — the key is making the reward feel earned, not trivial. The animations on badge unlock were carefully tuned to feel satisfying.

Gamification
05
PERSISTENT DATA VIA LOCALSTORAGE

All streaks, tasks, and session history are stored in localStorage — no backend, no account needed. This was a deliberate choice: a productivity tool that requires signup creates friction before value. Users should get value instantly.

Technical Decision
04 Technical Stack

BUILT LEAN, RUNS FAST

Focus Flow is built entirely with vanilla web technologies — no frameworks, no build tools. This was a conscious decision: the fewer dependencies, the faster the load, the more control I have over every interaction.

HTML5 CSS3 + Variables Vanilla JS (ES6+) Spotify IFrame API HTML5 Drag & Drop LocalStorage API CSS Animations Canvas API

The timer uses setInterval with drift correction (compensating for browser throttling of inactive tabs), and all session analytics are computed client-side.

06 What I Learned

LESSONS CARRIED FORWARD

✦ Constraints breed creativity. Building without frameworks forces you to understand the platform deeply — I now have a much clearer mental model of the browser's event loop and rendering pipeline.
✦ Animation budget matters. I initially had too many micro-animations firing simultaneously, which caused jank on lower-end devices. Learning to profile and batch animations was a crucial performance lesson.
✦ Gamification is subtle. Heavy-handed reward systems feel hollow. The most effective elements were small — a 2px progress line filling up, a number ticking over. Restraint makes rewards feel earned.
✦ Ship early, iterate. The first version had no Spotify, no analytics, no streak system. Each feature was added based on actual use — not upfront design. Shipping early gave me real feedback loops.
